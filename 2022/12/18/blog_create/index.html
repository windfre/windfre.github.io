My First Blog 小白超详细个人博客创建教程（GitHub+hexo)

希望对想建立个人博客的小伙伴儿有所帮助哦!

准备工作

1. 注册github账号
2. 下载安装git和nodejs

准备工作注意事项

  1. 注册GitHub账号 （参考链接：GitHub注册教程（图文详解））

    1. 使用Fast GitHub进行GitHub加速（参考链接: GitHub加速神器：Fast GitHub使用方法）

      1. 项目下载百度网盘链接地址：
        链接：https://pan.baidu.com/s/1EBYUEI17VmtdNH-c9lyRAA
        提取码：2bp7
      2. 安装及使用
        解压下载的zip包，将文件夹放到合适位置
        找到FastGithub.UI.exe, 双击执行即可
      3. 注意
        在使用GitHub前需要打开并保证Fast GitHub在运行状态
      4. 在安装完成之后可以直接点开fastgithub的执行文件就可以运行程序，顺利连接到Github。
      5. 如果不能正常打开.exe的执行文件的话还是使用命令去启动

    2. 进入GitHub官网 https://github.com

    3. 点击绿色框Sign up for GitHub进行注册

    4. 注册页面第一栏email(邮箱)，第二栏password(密码)，第三栏username(用户名)进行验证后即可完成注册。

  1. git的下载及安装 （参考链接 Windows系统Git安装教程（详解Git安装过程））(超级详细版参考链接 Git 详细安装教程（保姆级详细教程）)
    1. 下载地址 https://git-scm.com/downloads
    2. Git官网会自动识别当前使用的操作系统，所以右侧直接显示下载使用Windows系统的最新版本（如果识别错误，可以在中间选择系统），点击即可下载。下载到电脑上之后，鼠标双击文件即可进入安装过程。
    3. 使用许可声明，点击next
    4. 选择安装路径

      在输入框内输入想要安装到的本机路径，也就是实际文件夹位置，或点击“Browse…”选择已经存在的文件夹，然后点击“Next”按钮继续，进入下图界面：
    5. 选择安装组件

      选择默认即可，点击next。
    6. 选择开始菜单页

      这个界面是创建开始菜单中的名称，不需要修改，直接点“Next”按钮继续到下图的界面：
    7. 选择Git文件默认的编辑器

      这个页面是在选择Git文件默认的编辑器，很少用到，所以默认Vim即可，直接点“Next”按钮继续到下图的界面：
    8. 决定初始化新项目(仓库)的主干名字

      第一种是让 Git 自己选择，名字是 master ，但是未来也有可能会改为其他名字；第二种是我们自行决定，默认是 main，当然，你也可以改为其他的名字。一般默认第一种，点击 [next] 继续。
    9. 调整你的 path 环境变量

      第一种是仅从 Git Bash 使用 Git。这个的意思就是你只能通过 Git 安装后的 Git Bash 来使用 Git ，其他的什么命令提示符啊等第三方软件都不行。
      第二种是从命令行以及第三方软件进行 Git。这个就是在第一种基础上进行第三方支持，你将能够从 Git Bash，命令提示符(cmd) 和 Windows PowerShell 以及可以从 Windows 系统环境变量中寻找 Git 的任何第三方软件中使用 Git。推荐使用这个。
      第三种是从命令提示符使用 Git 和可选的 Unix 工具。选择这种将覆盖 Windows 工具，如 “ find 和 sort ”。只有在了解其含义后才使用此选项。一句话，适合比较懂的人折腾。
    10. 选择 SSH 执行文件

      是否选择安全的连接，在这里我们选择是，即默认第一个。
    11. 选择HTTPS后端传输

      作为普通用户，只是用 Git 来访问 Github、GitLab 等网站，选择前者就行了。点击 [next] 继续。
    12. 配置行尾符号转换 

      选择第一项，签出 Windows 样式，提交 Unix 样式的行结尾。点击[next] 继续。
    13. 配置终端模拟器以与 Git Bash 一起使用

      建议选择第一种，MinTTY 3功能比 cmd 多，cmd 只不过 比 MinTTY 更适合处理 Windows 的一些接口问题，这个对 Git 用处不大，除此之外 Windows 的默认控制台窗口（cmd）有很多劣势，比如 cmd 具有非常有限的默认历史记录回滚堆栈和糟糕的字体编码等等。
      相比之下，MinTTY 具有可调整大小的窗口和其他有用的可配置选项，可以通过右键单击的工具栏来打开它们 git-bash 。点击 [next] 继续。
    14. 选择默认的 “git pull” 行为
    15. 选择一个凭证帮助程序 

      第一个选项是提供登录凭证帮助的，Git 有时需要用户的凭据才能执行操作；例如，可能需要输入用户名和密码才能通过 HTTP 访问远程存储库（GitHub，GItLab 等等）。
    16. 配置额外的选项 

      第一项，启用文件系统缓存将批量读取文件系统数据并将其缓存在内存中以进行某些操作（”core.fscache” 设置为 “true”）。这可以显着提高性能。
    17. 配置实验性选项

      不建议配置，点击install进行安装。
    18. 安装结束

    19. 安装成功

      Git Bash： Unix与Linux风格的命令行，使用最多，推荐最多
      Git CMD： Windows风格的命令行
      Git GUI： 图形界面的Git，不建议初学者使用，尽量先熟悉常用命令
  1. Nodejs的下载及安装 Nodejs详细安装教程
    1. 下载地址 https://nodejs.org/en/
    2. 点击下载红色方框版本
    3. 双击进入安装页面

      点击next继续
    4. 同意许可声明，点击next继续
    5. 更改存储路径（也可按默认）
    6. 选择安装模式，无特殊需求，直接选择Node.js runtime
    7. 不用选择安装其他的插件（如果选择会消耗2.3g流量的）
    8. 点击install进行安装
    9. 检查是否安装成功
      安装好后在本机终端中测试是否成功如果出现版本号就说明成功了
      使用快捷键 win+r => 输入cmd 指令进入命令提示符窗，输入以下指令 能正确显示版本号，证明安装成功。
